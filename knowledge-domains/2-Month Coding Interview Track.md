# 2-Month Coding Interview Track

从算法基础到面试高分表现的结构化学习路径。适合准备技术面试，或希望系统提升算法和数据结构能力的开发者。

## Target Audience
- **Experience Level**: 0-5年工作经验
- **Current Role**: 软件工程师、学生、转行者
- **Goals**: 通过技术面试、提升算法能力、准备LeetCode

## Learning Objectives

完成本track后，你将能够：
- 🎯 解决常见算法问题类型
- 🎯 分析时间和空间复杂度
- 🎯 清晰表达解题思路
- 🎯 在白板或编辑器中实现解法
- 🎯 通过大型科技公司的coding面试

## Daily Study Schedule

### Weekday (2-3小时)
**Pattern Learning (1小时)**:
- 30分钟：阅读pattern和理论
- 30分钟：理解模板和例题

**Problem Practice (1.5小时)**:
- 15分钟：独立思考
- 30分钟：实现解法
- 30分钟：分析optimal solution
- 15分钟：总结和笔记

**Review (30分钟)**:
- Review之前学过的patterns
- 整理错题和难点

### Weekend Day (4-6小时)
**Morning Session (2-3小时)**:
- 集中练习新pattern
- 完成5-8道相关题目
- 深入理解复杂度分析

**Afternoon Session (2-3小时)**:
- Mock interview练习
- Review本周所有题目
- 准备下周内容

## Common Pitfalls to Avoid

### 1. Jumping to Coding Too Fast
❌ **错误**: 马上开始写代码
✅ **正确**: 先clarify，再设计，最后实现

### 2. Ignoring Edge Cases
❌ **错误**: 只考虑正常输入
✅ **正确**: 主动考虑empty、null、boundary等cases

### 3. Not Communicating
❌ **错误**: 默默思考不说话
✅ **正确**: 边想边说，让面试官follow思路

### 4. Giving Up on Hard Problems
❌ **错误**: 遇到困难就放弃
✅ **正确**: 展示思考过程，讨论approaches
