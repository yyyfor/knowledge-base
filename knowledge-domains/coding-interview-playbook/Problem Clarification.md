# Problem Clarification

题目澄清决定你是不是在解正确的问题，因为很多 coding 题的难度差异就藏在约束和边界里。

## Notes

面试官通常不会把所有条件一次性说完，所以高质量回答要主动把输入规模、输入特征、修改输入权限、空间限制和目标复杂度问出来。你问的问题越准，后面的模型和优化就越稳。

澄清不是拖时间，而是避免在错误假设上走远。例如“输入是否有序”“是否允许重复”“是否一定存在解”这类问题，经常会直接决定你用 two pointers、hashmap 还是 binary search。

## Key Points

- ask for input size and constraints first
- clarify whether the input is sorted, duplicated, or may contain negative values
- ask whether an optimal solution is required
- ask whether you can modify the input in place
- ask whether memory usage is constrained
