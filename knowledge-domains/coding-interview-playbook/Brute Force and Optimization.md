# Brute Force and Optimization

先说 brute force 再优化，不是浪费时间，而是在向面试官展示你知道正确性从哪里来、复杂度为什么值得优化。

## Notes

很多候选人最大的问题不是不会最优解，而是直接跳到最优解，导致面试官听不到推导过程。一个稳定的节奏应该是先给 baseline，说明它为什么正确，再明确指出瓶颈，然后自然过渡到优化版。

这一步会直接影响面试官的安全感。你说得出 “A brute force approach would be...” 和 “But this is too slow, so we need to optimize.”，就等于把思路分层了。

## Key Points

- state the brute force solution first
- explain why brute force is correct before saying it is too slow
- name the bottleneck explicitly
- connect the optimization to a specific data structure or invariant
