# Coding Interview Playbook

Coding interview 的高分主线不是“会不会写题”，而是能否按 understand → model → solve → optimize → defend 的顺序稳定推进。

## Notes

这类面试真正考的是你如何把一个模糊问题快速转成清晰模型，再在有限时间里给出正确、可解释、可优化的方案。真正拉开差距的通常不是最后那几行代码，而是你在写代码前是否先把约束、思路、复杂度和边界讲清楚。

一个稳的答题节奏通常是：先澄清 input、constraints 和 edge cases；再把题目映射到熟悉模型；先给 brute force 建立安全感；然后优化到目标复杂度；最后用 complexity、edge case 和 follow-up 把答案守住。

## Key Points

- start with clarify before coding
- model the problem before choosing data structures
- state a brute force baseline before jumping to the optimal solution
- explain why the optimized approach works
- 主动给出 time complexity 和 space complexity
- 用 edge cases 和 follow-up 把答案 defend 完整
