# Coding Communication

Coding 阶段真正被考察的不是打字速度，而是你能否把思路稳定翻译成面试官听得懂的代码。

## Notes

面试官通常不会逐字符检查你写得多快，他们更在意你是否持续同步自己的意图、变量含义和状态变化。边写边说可以让对方知道你没有失去控制，也能更早发现误解。

clean code 不是锦上添花，而是沟通的一部分。命名清晰、逻辑分段、条件判断明确，都会让你的思路显得更可信。很多时候，代码结构本身就是解释的一部分。

## Key Points

- talk while you code
- use clear names for pointers, counters, and maps
- explain initialization, iteration, and update steps
- keep the code aligned with the model you just described
