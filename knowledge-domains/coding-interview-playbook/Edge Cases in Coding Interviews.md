# Edge Cases in Coding Interviews

Edge case 不是最后随便补两句，而是检验你的算法是否真正稳固的最后一关。

## Notes

很多题挂掉并不是因为核心思路错了，而是因为空输入、单元素、重复元素、负数、越界、no-solution 这类边界没有提前纳入设计。真正稳定的候选人会把 edge case 当作建模的一部分，而不是写完代码才想起来。

一个成熟的习惯是写代码前先口头列出最危险的边界，写完后再用这些边界走一遍代码。这能同时帮助你发现 bug，也能让面试官看到你有交付意识。

## Key Points

- empty input
- single element
- duplicates
- negative numbers
- no solution or multiple valid solutions
