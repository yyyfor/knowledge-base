# Common Coding Interview Mistakes

很多 coding interview 的失分点并不高级，反而是最基础的流程错误：没澄清、没建模、没复杂度、没边界。

## Notes

最危险的错误通常包括一上来写代码、不解释复杂度、不主动过 edge case、以及思路跳跃太快。它们的共同问题是：面试官看不到你的推理过程，只能看到一个可能对也可能错的实现。

这些错误之所以高频，是因为候选人把 coding interview 误解成 speed contest。实际上，结构化表达和推导过程往往比最终代码快几分钟写完更重要。

## Key Points

- 不要一上来就写代码
- 不要省略复杂度分析
- 不要忽略 edge case
- 不要让思路跳跃到面试官跟不上
