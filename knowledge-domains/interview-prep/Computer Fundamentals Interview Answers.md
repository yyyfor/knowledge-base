# Computer Fundamentals Interview Answers

计算机基础题通常不是考你背概念，而是看你能不能把数据结构、复杂度、编码、内存、进程线程和 I/O 模型放到真实后端系统里理解。

## 进程和线程有什么区别？

参考回答：

进程是资源分配的基本单位，有独立地址空间、文件句柄和运行环境。线程是 CPU 调度的基本单位，同一进程内的线程共享地址空间和资源，所以线程间通信成本低，但也更容易出现数据竞争。

面试里要补工程影响：多进程隔离性更好，一个进程崩溃不一定影响另一个；多线程更轻量，但需要处理锁、可见性、死锁和线程池资源隔离。

## 用户态和内核态有什么区别？

参考回答：

用户态运行普通应用代码，不能直接访问硬件和关键内核资源；内核态运行操作系统内核代码，可以执行特权指令。应用进行文件、网络、进程调度等操作时，需要通过系统调用从用户态切到内核态。

系统调用和上下文切换有成本，所以高性能系统会尽量减少频繁切换，比如批量 I/O、零拷贝、连接复用和异步 I/O。

## 上下文切换为什么有成本？

参考回答：

上下文切换需要保存当前线程或进程的寄存器、程序计数器、栈等状态，再恢复另一个执行单元的状态，还可能造成 CPU cache/TLB 失效。线程太多或锁竞争严重时，CPU 时间会花在调度上，而不是业务计算上。

这也是为什么线程池不能无限加线程。IO-bound 可以比 CPU core 多一些，但必须受下游容量和队列长度约束。

## 什么是时间复杂度和空间复杂度？

参考回答：

时间复杂度描述输入规模增长时算法执行步骤的增长趋势，空间复杂度描述额外内存使用增长趋势。它们不是精确耗时，而是增长阶。

工程里复杂度仍然重要。比如 O(n²) 在小数据上可以接受，但在接口每秒处理大量请求时会迅速放大 CPU；空间复杂度高可能导致 GC 压力和内存峰值。

## Hash 为什么能做到接近 O(1) 查找？

参考回答：

Hash 表通过 hash 函数把 key 映射到数组位置，理想情况下可以直接定位桶，所以查找接近 O(1)。但如果 hash 冲突严重，需要在桶内继续比较，复杂度会退化。

这就是为什么 key 的 `hashCode()` 质量、不可变性和 equals/hashCode 一致性很重要。HashMap 的性能不是魔法，它依赖均匀分布和合理扩容。

## Unicode、UTF-8、ASCII 有什么区别？

参考回答：

ASCII 是早期字符编码，只覆盖有限英文字符。Unicode 是字符集，给不同语言字符分配统一码点。UTF-8 是 Unicode 的一种变长编码方式，英文通常 1 字节，中文通常 3 字节，兼容 ASCII。

线上常见问题是字符集不一致导致乱码，或者按字节截断字符串导致多字节字符损坏。接口、数据库、消息队列和文件处理要统一编码。

## 大端和小端是什么？

参考回答：

大端和小端描述多字节数据在内存中的字节顺序。大端把高位字节放在低地址，小端把低位字节放在低地址。

普通业务代码很少直接处理，但网络协议、二进制文件、跨语言序列化、底层存储和高性能通信会遇到。面试里知道它影响二进制解析和协议兼容即可。

## 什么是序列化？为什么要关注兼容性？

参考回答：

序列化是把对象转换成可存储或传输的格式，反序列化是恢复对象。兼容性问题来自字段新增、删除、重命名、类型变化和不同语言实现差异。

生产系统里消息协议要支持 schema 演进，例如字段默认值、向前/向后兼容、版本号和灰度发布。否则新旧服务同时运行时可能解析失败。

## CPU-bound 和 IO-bound 任务有什么区别？

参考回答：

CPU-bound 主要消耗 CPU，例如加密、压缩、复杂计算，线程数通常接近 CPU core。IO-bound 主要等待外部资源，例如数据库、网络、文件，线程数可以更多，但要受下游容量限制。

线程池配置要基于任务类型，不要所有任务共用一个池。否则慢 IO 任务可能占满线程，拖垮 CPU 任务或核心链路。

## 面试收尾句

计算机基础不是孤立概念。进程线程影响并发模型，系统调用影响 I/O 成本，复杂度影响吞吐，编码影响数据正确性，序列化影响服务兼容。回答时最好都能落到具体工程后果。
