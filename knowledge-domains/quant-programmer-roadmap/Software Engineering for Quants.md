# Software Engineering for Quants

量化程序员不是只写 notebook，而是要交付可维护的软件。

详细解释：

核心能力包括测试、版本管理、代码评审、接口设计、可观测性和生产稳定性。很多 quant 团队的问题不在模型本身，而在工程质量太差，导致结果不可复现、不可审计、难以协作。

重点主题：

- testing
- version control
- API design
- observability
- reproducibility
